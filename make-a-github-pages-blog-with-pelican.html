Make a Github Pages blog with Pelican

  • Nov 24, 2024
  • Categories: Python
  • Tags: #python

Install Pelican

In project’s folder:

pip install pelican

Create a new folder for your project and open a console in it. Let's create default Simple theme:

pelican-quickstart

During the pelican-quickstart process you need to say "yes":

 Do you want to upload your website using Github Pages? Say “y”

 Do you want to automate site generation and publication? Say "y"

If you answered “yes” to second question, a tasks.py and Makefile will be generated in the root of your project. These files should be customized to fit your particular needs and usage patterns. Using them you can easily view and publish your site.

Theme installation

You can find pelican themes here.Be carefull: not all themes are new and modern looking.

Navigate to project's root folder and clone your theme. 

git clone https://github.com/aleylara/Peli-Kiera

Install and verify the installation (stay in progect's root folder): 

pelican-themes -i Peli-Kiera
pelican-themes --list –v

The last command shows the path to your theme.

Configure blog to use new theme

For this, just open your pelicanconf.py available in your root folder and define the path to your new theme by adding the following statement at the end of pelicanconf.py file:

THEME = '/usr/local/lib/python3.11/dist-packages/pelican/themes/Peli-Kiera'

If your theme have own pelicanconf.py you need extra steps. In project's root folder find Makefile, open it in your favorit editor and put correct paths to theme's content, output folder and settings. For example:

BASEDIR=$(CURDIR)
INPUTDIR=$(BASEDIR)/your_theme/content
OUTPUTDIR=$(BASEDIR)/your_theme/output
CONFFILE=$(BASEDIR)/your_theme/pelicanconf.py
PUBLISHCONF=$(BASEDIR)/your_theme/publishconf.py

Then in project's root folder open tasks.py and change values in variables:

SETTINGS_FILE_BASE = 'your_theme/docs/pelicanconf.py'
...
'settings_publish': 'your_theme/docs/publishconf.py'

If you will have problems with rendering pages because of CACHE try open another page of site and then return to previous page again. IF you still have problems with CACHE edit pelicanconf.py in theme's folder: 

LOAD_CONTENT_CACHE = False
AUTORELOAD_IGNORE_CACHE = True

You can add default pagination count. You need put "0" here to show tags button on the main page.

DEFAULT_PAGINATION = 0

Creating blog content in Markdown

In project's root folder navigate to content folder (create content folder if not exist). In content folder create folders: articles (for blog posts) and pages (for static pages). Install markdown and create files.md in both folders.

A sample article header.

Title: Articles_title 
Date: 2024-11-14
Tags: python
Description: The Zen of Python
Category: Python

It's time to view our Simple Theme. Navigate to project's root folder and run:

make devserver

This step create OUTPUT folder with site files in root of your project. This command also tell Pelican to watch for your modifications, instead of manually re-running it every time you want to see your changes. In Linux it simultaneously both auto-regenerate and serve the output at http://127.0.0.1:8000/ or http://localhost:8000 (depends on your theme settings).

Go to http://127.0.0.1:8000 or http://localhost:8000/ in your browser to view your blog. If you want to stop development server run:

Ctrl+C

When you are ready to deploy your site for production change SITEURL in theme's pelicanconf.py from: 

SITEURL = "http://localhost:8000"

to

SITEURL = "https://your_site.github.io"

Stop development server and run it again:

make devserver

Push site on github

Create repository for your Github Pages on Github. Clone your site folder:

git clone https://github.com/paths13/paths13.github.io.git

In cloned folder:

git init .

You should see .git folder. 

git checkout -b main

Copy files in OUTPUT folder of your theme (in Flex-master folder) and paste it into cloned site folder from github.

git add .

git commit -m "initial commit"

git push https://your_token@github.com/paths13/paths13.github.io.git

You are all done!

Note! Since 2022 you need to receive token instead of login and password to push site on github.
